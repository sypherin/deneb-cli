    touch the write tier or guards. (nvidia-smi, rocm-smi, lspci, sysctl are already allowlisted;

    In deneb/hardware.py add the LIVE layer beneath the pure parsers (keep the pure section
    I/O-free). Write one thin, INDEPENDENT probe function per vendor, each of which: calls
    tools.run(...) for its exact command, then hands the captured output string to the matching
    pure parser, and returns [] on absence/failure — a vendor path must never raise and must
    never block another vendor:
      - _probe_nvidia(): tools.run("nvidia-smi --query-gpu=name,memory.total,driver_version
        --format=csv,noheader") -> parse_nvidia_smi; backend "cuda".
      - _probe_amd(): tools.run("rocm-smi --showproductname --showmeminfo vram --json") ->
        parse_rocm_smi; backend "rocm"; best-effort driver via tools.run("rocm-smi
        --showdriverversion") parsed loosely (optional, tolerate absence).
      - _probe_apple(): only attempt on Darwin; tools.run("system_profiler SPDisplaysDataType")
        -> parse_system_profiler, and tools.run("sysctl hw.memsize") -> parse_sysctl_memsize for
        the unified pool; backend "metal".
      - _probe_lspci_fallback(): tools.run("lspci -nn") -> parse_lspci_gpus; used ONLY to name a
        GPU whose vendor SMI tool is absent (so the profile can say "NVIDIA GPU present, nvidia-smi
        not installed — VRAM unread"), never overriding a real SMI reading.
    Treat the tools.run sentinels "(not installed on this box: ...)" and "[refused] ..." as
    absence (the parsers already do; the probe just returns []).

    Implement profile_hardware() -> HardwareProfile: derive os/arch/kernel from platform (system,
    machine, release); cpu from parse_cpuinfo(tools.read_file("/proc/cpuinfo")) on Linux, else
    platform.processor()/sysctl on Darwin; ram_total_mb from parse_meminfo(read_file
    "/proc/meminfo") on Linux, parse_sysctl_memsize on Darwin; disk_free_mb from
    shutil.disk_usage("/").free // (1024*1024) (stdlib, no probe needed). Run ALL vendor probes
    independently, concatenate their GPUInfo lists, plus the lspci fallback for any vendor with a
    GPU-but-no-SMI. For each GPU call classify_memory(gpu, ram_total_mb). Set primary_backend to
    the first present GPU backend, else "cpu". Set profile.usable_mem_mb from the primary GPU's
    usable_mem_mb, else ram_total_mb minus the documented unified headroom (CPU-only box uses RAM).
    Never let one failing probe abort the whole profile — wrap each probe call so an unexpected
    error becomes [] + a note, mirroring check.py's per-probe try/except (a probe error is a
    degraded field, never a crash).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| vendor tool subprocess -> profile_hardware | live tool stdout crosses into the profile; must degrade, never crash |
| tools.py allowlist expansion | adding system_profiler widens the read-only boundary by one macOS info tool |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Denial of Service | one vendor probe failing aborts the whole profile | mitigate | each probe wrapped in try/except returning [] + a note (check.py per-probe pattern); CPU-only + absent-tool tests prove profile still returns |
| T-02-02 | Elevation of Privilege | new allowlist entry enables a mutating command | mitigate | system_profiler is read-only with no write modes; runs via tools.run shell=False, no sudo; only the one info subcommand is used |
| T-02-03 | Information Disclosure | profile output leaks a secret | accept | GPU names / RAM / disk are non-sensitive; tools.run already redacts secret-looking tokens before output leaves the box |
| T-02-SC | Tampering | supply chain (npm/pip installs) | accept | no new packages — stdlib only |
</threat_model>
