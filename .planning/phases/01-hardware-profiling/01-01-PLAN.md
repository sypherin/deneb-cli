<objective>
Build the PURE, deterministic core of hardware profiling: the HardwareProfile data model,
one pure parser per vendor tool (NVIDIA / AMD / Apple) plus CPU/RAM parsers, and the
unified-memory + usable-memory classification logic — all unit-tested against captured real
tool outputs, with NO live tool invocation.

This is the interface-defining plan. It establishes the contracts (structs + parser
signatures) that Plan 02's live-probe layer will consume. Separating INVOCATION (Plan 02)
from PARSING (this plan) is the HW-04 / QA-01 testability requirement: parsers take captured
output strings and return structured data, tested with no live tool present.

Purpose: the pure layer is where every hardware fact is derived; it must be defensive
(never crash on junk) and it must not fall into the unified-memory trap (this box's rocm-smi
reports 1GB "VRAM" while actually carrying 128GB of shared memory).
Output: deneb/hardware.py (model + parsers + classification), tests/test_hardware.py, and a
tests/fixtures/hardware/ corpus of captured real + canonical sample outputs.
</objective>
