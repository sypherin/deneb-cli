<fit_math_spec>
<!-- Exact deterministic rules for Task 2 so tests can pin them. Prose only; no code fences in <action>. -->

RUNTIME_RESERVE_MB(size_mb) = max(512, int(0.10 * size_mb))  — a coarse KV/context/compute
allowance ON TOP of weights. Documented as coarse (refined by a later phase). This is separate
from the profile's OS headroom (already in usable_mem_mb) — no double counting.

need_mb = quant.size_mb + RUNTIME_RESERVE_MB(quant.size_mb).
headroom_mb = usable_mem_mb - need_mb  (only when usable_mem_mb is a real int).
fits = usable_mem_mb is not None AND headroom_mb >= 0.

Speed-tier heuristic (label): TIER_ORDER = ["very-slow", "slow", "medium", "fast"].
  effective_params = active_b if arch == "moe" and active_b else params_b.
  by effective_params: <=8 -> fast; <=20 -> medium; <=40 -> slow; else very-slow.
  backend adjust: primary_backend == "cpu" downgrades the tier by 2 (floor very-slow);
  cuda/rocm/metal are accelerated and neutral (coarse: cuda is the fastest accelerator but at this
  tier granularity cuda and rocm share a tier — ROCm-specific risk is surfaced via caveats, not the
  tier). This makes a 35B-A3B (effective 3) "fast" on cuda AND rocm — the canonical case.

Caveat rules (FIT-02b) — return caveats: list[str], any that apply, deterministic order:
  - gfx1151/ROCm: if any gpu.extra.get("gfx") in {"gfx1151","gfx1150"} (or backend rocm on a
    known-caveat gfx) -> a note that Strix Halo/gfx1151 needs a recent kernel with the ROCm fixes;
    older kernels have known gfx1151 instability.
  - unified shared pool: if the primary GPU memory_kind == "unified" -> warn the model shares the
    one memory pool with the OS/apps; close other heavy apps.
  - tight fit: if fits AND headroom_mb < max(2000, int(0.10 * need_mb)) -> warn it is a tight fit
    (~headroom_mb MB spare); expect reduced context length / possible OOM under load.
  - cpu-only: if primary_backend == "cpu" -> warn CPU-only means low tok/s; prefer a smaller model.
  - unknown budget: if usable_mem_mb is None -> a caveat that the box's memory budget was
    unreadable, so fit could not be computed (fits=False, honest degradation).

FitResult dataclass fields: fits: bool, headroom_mb: int, expected_speed_tier: str,
caveats: list[str], need_mb: int. (need_mb is extra context for ranking/why in Plan 02; the
{fits, headroom_mb, expected_speed_tier, caveats} contract from FIT-02 is fully present.)
</fit_math_spec>

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02-SC | Tampering | supply chain (pip installs) | accept | no new packages — stdlib dataclasses + already-present pytest only; nothing to audit |
